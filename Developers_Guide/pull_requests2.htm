CLIMSOFT 4 - Developer’s guide

Creating a `Pull Request`


A `Pull request` is used to merge your changes into the main project.  First we will create a new branch for your changes to be submitted on.

1.       Open Github for windows

2.       Select the `master` branch (always the latest version of the code)

3.       If there are updates on the server then click the `sync` button to receive these

4.       Create a new branch for your update: click the drop down next to `master` and enter the name for you new branch in the `Filter or create new` box and click create

5.       The select branch will now show the new name (instead of `master`)

6.       Right click on the `Climsoft` repository on the left hand side and choose `Open in Explorer`

7.       Add the file that you wish to contribute to the repository folder

In this example we replaced `mysql_climsoft_db_v4.sql` in the `Climsoft_V4_dbase` folder

8.       Github for windows indicates that we now have `Uncommitted changes`

9.       Click on the drop down arrow next to `Show` to see the files you have changed

For each file, lines that were removed are shown in red, whereas new lines that you have added are shown in green.

10.   Type a short description into the `Summary` box and click on the Commit button to save your changes

11.   Your changes are committed, but they are not yet synchronised with github.com (this is indicated by the `Unsynced changes` message.  Click `Publish` to upload this new branch to github.com

Note that you also have the option to `Undo` the commit – this will not delete the files you have changed, it will just remove the decision to commit this change into source control

 

12.   Now that your branch is published, click to see it on github.com

 

13.   This page is showing a single commit. Click on the name of your branch to see the branch overall

14.   The page that opens allow you to `Create pull request` that will allow your work to be merged into `master`.

15.   On this `Open a pull request` page you have the option to write a description before clicking. `Create pull request`.  Note that `base: master` indicates where your branch will be merged.

 

Reviewing and merging a `Pull Request`


A list of current pull requests can be viewed at https://github.com/opencdms/Climsoft/pulls

Select the pull request that you would like to review and merge.

 

1.       First click on `Files changed` to see the changes that are being submitted

2.       If you have an questions you can discuss these via the conversation tab.  Type in the `Leave a comment` section and click `Comment`

3.       To accept the pull request, click `Merge pull request`.  To reject it, type the reason into the comment box and click `Close pull request`

REJECT
 

4.       Selecting `Merge pull request` will open a final box where you can click `Confirm merge`

5.       The page updates to show that the pull request has been merged
